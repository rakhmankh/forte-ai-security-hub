# Forte AI Security Hub

**Интеллектуальная система выявления мошенничества в мобильном банкинге нового поколения.**

Разработано командой **keeyman** в рамках **AI Hackathon ForteBank**.

---

## О проекте
**Forte AI Security Hub** — это не просто ML-модель, а комплексная экосистема безопасности, объединяющая передовые алгоритмы машинного обучения с удобными инструментами для бизнеса.

Мы решаем проблему «черного ящика» в антифроде: наша система не просто блокирует транзакции, но и **объясняет причины** каждого решения на понятном языке, помогая сократить время расследования инцидентов и сохранить лояльность клиентов.

### Ключевые преимущества (Business Value)
* **Real-time Scoring:** Мгновенный анализ транзакций и блокировка угроз в реальном времени.
* **Explainable AI (XAI):** Использование **SHAP values** для интерпретации решений. Оператор видит, *почему* транзакция помечена как фрод (например, *"Аномальная сумма для ночного времени"*).
* **Data Replay (Эмуляция):** Уникальная возможность проигрывания исторических данных банка для нагрузочного тестирования и демонстрации.
* **Управление рисками:** Дашборд с метриками в реальном времени (количество атак, сумма предотвращенного ущерба в KZT).
* **Автоматизация:** Генерация PDF-протоколов инцидентов одной кнопкой.
* **MLOps Ready:** Встроенный пайплайн для дообучения модели на новых данных (Feedback Loop).

## Технологический стек

* **ML Core:** `CatBoost` (обучен на реальных исторических транзакциях), `SHAP` (интерпретация), `Scikit-learn`.
* **Backend:** `FastAPI` (Python 3.9), `AsyncIO` — для высокой производительности.
* **Frontend:** HTML5 / CSS3 / JavaScript (Vanilla) + `Chart.js` для аналитики.
* **Deployment:** `Docker`, `Docker Compose`.
* **Data Processing:** `Pandas`, `NumPy`.

## Установка и запуск

### Вариант 1: Запуск через Docker (Рекомендуемый)
Самый быстрый способ развернуть решение.

1.  Клонируйте репозиторий:
    ```bash
    git clone <ссылка на ваш репозиторий>
    cd forte_hackathon
    ```
2.  Запустите контейнер:
    ```bash
    docker-compose up --build
    ```
3.  Откройте приложение в браузере: [http://localhost:8000](http://localhost:8000)

### Вариант 2: Локальный запуск (Python)

1.  Установите зависимости:
    ```bash
    pip install -r requirements.txt
    ```
2.  **Подготовка данных:**
    Обработайте сырой датасет банка для обучения модели:
    ```bash
    python src/prepare_data.py
    ```
    *(Скрипт создаст файл `data/transactions_train.csv` на основе реальных данных)*.
3.  Обучите модель:
    ```bash
    python src/train.py
    ```
4.  Запустите сервер:
    ```bash
    python src/main.py
    ```
5.  Откройте дашборд: [http://localhost:8000](http://localhost:8000)

## Функциональность платформы

### 1. Сканер (Scanner)
Ручная проверка подозрительных операций.
* Ввод параметров транзакции (сумма, время, устройство).
* Мгновенный вердикт: **BLOCK** / **OK**.
* Детализация факторов риска (SHAP).
* Скачивание **PDF-отчета** об инциденте.

### 2. Пакетная проверка (Batch Processing)
Эмуляция потоковой обработки данных.
* Нажмите **"Эмуляция потока данных"**, чтобы запустить проверку на реальных исторических данных банка.
* Система обрабатывает пакет транзакций и выводит результаты в таблицу.

### 3. Центр мониторинга (Admin Dashboard)
Аналитика для бизнеса.
* График подозрительной активности в реальном времени.
* Счетчики: Всего транзакций, Выявлено атак, **Предотвращенный ущерб (в тенге)**.
* Кнопка **"Запустить дообучение"** — демонстрация MLOps процесса адаптации модели.
